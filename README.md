# OntoShip

> **A project knowledge base is just `markdown` + a `README` index + `git`.** Nothing simpler works better.

![OntoShip graph of a real project](docs/img/graph.png)

*Above: the actual documentation graph of a live production project
([hub.neuraldeep.ru](https://hub.neuraldeep.ru/)) ‚Äî generated by `gitmark map`, gravity

---

## The idea

Markdown is the source of truth ‚Äî the thing you and your AI assistant read directly, in
any editor. Everything **derived** (a search index, this HTML overview, the link graph) is
**regenerated** from the md, so git stays clean. No service, no database, no embeddings,
no vendor. The entry point is `CLAUDE.md` / `AGENTS.md`; every folder's `README.md` is its
index.

```
CLAUDE.md / AGENTS.md      ‚Üê entry point
/docs
  README.md                ‚Üê master index
  /services
    /<service>
      README.md            ‚Üê always the folder index
```

Battle-tested over months of building with AI assistants. The KB an agent (and a human)
actually wants is plain md you can read anywhere ‚Äî *plus* instant search so you find the
right file instead of grepping blind.

## What's in here

**`gitmark`** ‚Äî one self-contained Python-stdlib CLI, zero dependencies:

| command | what |
|---|---|
| `index` | build a SQLite **FTS5** index: `bm25` ‚à™ `trigram` (substring) ‚à™ fuzzy (4-char windows, typo/morphology/non-Latin tolerant) |
| `search` | ranked `file:line ¬∑ heading ¬∑ snippet`; `--json` for tooling |
| `map` | self-contained **HTML**: collapsible tree + rendered markdown + a force/radial **graph** of links |
| `serve` | tiny local HTTP server to view the map |
| `stat` | index/KB stats |
| `lint` | *(optional)* ontology checks: broken links, frontmatter, folder READMEs |
| `version` | ‚Äî |

Plus a **Claude Code plugin** with three skills and a `/kb` command:

| skill | what |
|---|---|
| `kb-search` | search the KB (FTS5) instead of grepping blind |
| `kb-curate` | light ontology rules when adding/editing docs (types, frontmatter, typed links) |
| `dev-flow` | the spec-driven loop to ship a feature: research ‚Üí tasks ‚Üí goal ‚Üí **spec (md via `kb-curate`)** ‚Üí isolated **git worktree** ‚Üí implement ‚Üí tests ‚Üí independent review ‚Üí dev-tests ‚Üí prod-tests ‚Üí ship (MR ‚Üí `dev` ‚Üí `main`). Feature to prod in ~40 min ‚Äì 2 h. |

So the agent searches the KB instead of grepping, follows light curation rules when
editing, and ships changes through one repeatable, gated flow built around the KB.

## Quickstart

```bash
python3 skills/kb-search/gitmark.py index            # build .gitmark/index.db
python3 skills/kb-search/gitmark.py search "auth flow"
python3 skills/kb-search/gitmark.py map -o docs-map.html && open docs-map.html
```

Pure `python3` (‚â•3.7) with SQLite **FTS5**; the **trigram** tokenizer (SQLite ‚â• 3.34) adds
fuzzy/substring/non-Latin matching ‚Äî detected automatically, degrades gracefully. Add
`.gitmark/` to `.gitignore` ‚Äî it's a build artifact.

## Install as a Claude Code plugin

```text
/plugin marketplace add vakovalskii/ontoship
/plugin install gitmark@gitmark-marketplace
```

Or just copy `skills/` and `commands/` into your repo's `.claude/` ‚Äî project-local, no
marketplace needed.

## Optional: the ontology (only if you want guardrails)

The simple core needs **no frontmatter at all** ‚Äî `index`/`search`/`map` work on raw
markdown. If a team wants the KB not to rot as it grows, there's a light **code-ontology**
(inspired by [Palantir's Ontology](https://www.palantir.com/docs/foundry/ontology/overview)):
each doc gets a `node_type`, a few frontmatter properties, and typed links ‚Äî and
`gitmark lint` enforces it (no broken links, no orphans, every folder has a README index).
See [`docs/ontology.md`](docs/ontology.md). Use it or ignore it; the tool doesn't force it.

## Why this and not a wiki / Notion / vector-DB memory

Because the simplest thing wins: md you already read, git you already use, one stdlib file
you can audit in an afternoon. It runs offline, leaks nothing to a SaaS, and you own every
byte. There are a thousand "memory banks" ‚Äî this one's bet is **radical simplicity that you
can watch working on a real product**, not another service to run.
